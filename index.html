Overview:

Pictured is an 8-bit arithmetic logic unit (ALU).  The ALU takes two 8-bit words as input and can perform four different operations: add (A+B), subtract (A-B), arithmetic shift left (2A) or arithmetic shift right (A/2).  Each operation has an enable line that can be selected using a multiplexer.

Output:

The Redstone Lamps at the bottom of the picture are the output bits of the ALU.  Starting from the left, the first 8 lamps represent the 8-bit word result.  The remaining 4 lamps are the flag bits: negative, carry, overflow and zero.

The result word:

        The result word is a two’s complement number in binary.  The range of numbers that can be represented is from 1000 0000 through 0111 1111 (-128 through 127 in base 10).

Negative:

The negative bit is set if the most significant bit of the result is also set.  This is because the most significant bit of the answer also represents the sign of the number in two’s complement.

Carry:

Typically, the carry bit is useful if you need than one cycle through the ALU to complete a computation.  In the scope of this project, the carry bit is unnecessary but its implementation is still worth discussing.  The only time the carry flag will be set is if there is a carry left over from an addition operation.  For example:

          1111

1111 0000

0111 0000

  •             ________

        0110 0000

16 + 144 = 160

Notice the final 1 that doesn’t fit into the 8-bits.  In this example the carry bit would be set to indicate that we are carrying a 1 into the 256ths place.

Overflow:

In the ALU:

Overflow occurs when adding two positive numbers yields a negative result or when adding two negative numbers yields a positive result.  Overflow is detected if the carry in to the sign bit and the carry out of the sign bit are different.  In other words, the carry in to the sign bit and the carry out of the sign bit go through an XOR gate which controls whether or not the overflow bit is set.  Notice in the previous example that the overflow bit would not be set because a 1 is both carried into and carried out of the 128ths place.  Consider the following example:

In base 10, this example is 144 + 208 = -80.  This is a case where adding two positive numbers yields a negative result.  The answer is not valid that is this case the overflow bit would be set to indicate this condition.

Overflow in shift left:

        I implemented multiplication by two by bit shifting.  This operation is known as arithmetic shift left and is also capable of producing an overflow condition.  Overflow in arithmetic shift left occurs when the two most significant bits are different.  Consider the following example as an illustration:
